# ブラウザキャッシュ起因のエラー対処

## 問題の背景

Firebase Hosting Emulator でJSモジュールを配信する際、ブラウザがESモジュールを強力にキャッシュする。
ファイルを更新しても古いバージョンが参照され続け、`does not provide an export named` のようなエラーが発生する。

## 対処方針

### ブラウザサブエージェントでテストする場合

1. **エラーが ESモジュールのインポートに関するものか確認する**
   - `does not provide an export named` → **キャッシュ起因の可能性大**
   - `is not defined` / `TypeError` → **コードのバグの可能性**

2. **キャッシュ起因と判断した場合は深追いしない**
   - ファイルの実体を `grep_search` や `view_file` で確認し、export が存在するか検証する
   - 存在するならキャッシュ問題と断定してユーザーに報告、ブラウザの再テストに時間を浪費しない

3. **テスト前に Emulatorを再起動してもらう**
   - `firebase.json` に `Cache-Control: no-cache` を設定済みだが、Emulator再起動が必要

### やってはいけないこと

- キャッシュ問題を「コードのバグ」と誤認して何度もファイルを書き換える
- `?v=2` のようなキャッシュバスターをインポートパスに付与する（コードが汚れる）
- ブラウザサブエージェントを何度も再実行してキャッシュを突破しようとする
