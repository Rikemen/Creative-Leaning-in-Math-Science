# ⚾ Baseball Form Analyzer

## 1. プロジェクトの目的と背景

野球のピッチング動作は非常に複雑であり、独学でのフォーム改善には限界があります。

本プロジェクトは、**「PoseNetを用いた骨格抽出により、プロ選手の理想的なフォームと自分のフォームを視覚的に比較・分析できる環境」** を、手軽なWebアプリとして提供することを目的とします。

---

## 2. プロジェクト目標 (Goals)

- **視覚的な比較機能:** 自分とプロ選手の動画を重ね合わせ（オーバーレイ）、または並列（サイドバイサイド）で再生し、関節の角度や重心の移動を可視化する。
- **データ駆動のフィードバック:** 感覚値ではなく、PoseNetから得られる座標データに基づいたフォームの差異を提示する。
- **アクセスの容易さ:** インストール不要のWebアプリとして、スマホやPCのブラウザから即座に利用可能にする。

---

## 3. 主要な機能 (Features)

| 機能                       | 説明                                                                               |
| -------------------------- | ---------------------------------------------------------------------------------- |
| **ユーザー管理**           | Firebase Authenticationによるログイン、過去のフォームデータの保存管理              |
| **動画アップロード・保存** | 自分の投球動画をFirebase Storageへアップロード                                     |
| **骨格検知 (PoseNet)**     | ml5.jsを用い、動画内の投球動作から主要な関節点（肩、肘、手首、腰、膝、足首）を抽出 |
| **比較ビューワー**         | 理想の選手データと自分のデータを同期再生                                           |
| **描画インターフェース**   | p5.jsによる直感的なUIと、骨格線のリアルタイム描画                                  |

---

## 4. 技術スタック (Tech Stack)

| カテゴリ      | 採用技術          | 役割                                                          |
| ------------- | ----------------- | ------------------------------------------------------------- |
| **Frontend**  | p5.js             | キャンバス描画、UI要素、動画制御                              |
| **ML Engine** | ml5.js (PoseNet)  | ポーズ推定（骨格の座標データ取得）                            |
| **Backend**   | Firebase          | 認証(Auth)、動画/データ保存(Storage/Firestore)、公開(Hosting) |
| **Language**  | JavaScript (ES6+) | アプリケーションロジック全般                                  |

---

## 5. 成功の定義

- [ ] アップロードした動画から、ピッチングの主要なフェーズ（セットアップ、踏み込み、リリース）が正確にトラッキングできていること。
- [ ] 「理想の選手」の座標データと「自分」のデータを、時間軸を合わせて同期表示できること。
- [ ] Firebaseを介して、過去の自分との比較（成長記録）が可能であること。
- [ ] (Advanced)理想のフォームとの差異、どのような動作を改善すればいいのかをアドバイスしてくれるAIを実装すること

---

## 6. リスクと対策

| リスク                                                     | 対策                                                                                           |
| ---------------------------------------------------------- | ---------------------------------------------------------------------------------------------- |
| **処理負荷:** ブラウザ上での姿勢推定は重くなる可能性がある | フレームレートの制限や、解析済みの座標データをFirestoreに保存して再利用する                    |
| **同期の難しさ:** 選手によって投球テンポが異なる           | リリースポイントなどの特定のフレームを手動、または自動でマーキングして同期させる機能を検討する |
