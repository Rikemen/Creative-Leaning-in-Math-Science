# 🛠️ 開発ログ: Baseball Form Analyzer MVP

> 各ステップは **約10分** で完了する粒度に設計。  
> 各ステップ完了時にブラウザで動作確認し、動いたら次へ進む。

---

## Step 1: HTMLの骨格を作る（10分）

**ゴール:** ブラウザで空のページが表示される

- [ ] `index.html` を作成
- [ ] p5.js / ml5.js のCDNリンクを `<script>` で追加
- [ ] `style.css` を作成し、リンク
- [ ] `sketch.js` を空の `setup()` / `draw()` で作成
- [ ] ブラウザで開いて、真っ白なキャンバスが表示されることを確認

```
作るファイル: index.html, style.css, sketch.js
確認方法: ブラウザで index.html を直接開く
```

**✅ 完了条件:** コンソールエラーなしでキャンバスが表示される

---

## Step 2: 動画ファイルの読み込み（10分）

**ゴール:** ファイル選択から動画を読み込み、キャンバスに表示される

- [ ] `<input type="file" accept="video/*">` を追加
- [ ] `createFileInput()` または DOM の `change` イベントで動画ファイルを受け取る
- [ ] `createVideo()` で動画要素を生成
- [ ] 動画を非表示にし（`video.hide()`）、`image(video, ...)` でキャンバスに描画

```
確認方法: 適当な動画ファイルを選択 → キャンバスに最初のフレームが表示される
```

**✅ 完了条件:** ローカルの動画ファイルがキャンバス内に表示される

---

## Step 3: 再生/停止ボタン（10分）

**ゴール:** 動画の再生と停止をボタンで制御できる

- [ ] 再生/停止トグルボタンを `createButton()` で作成
- [ ] `video.play()` / `video.pause()` をボタンに紐付け
- [ ] ボタンのラベルを再生中は `⏸ 停止`、停止中は `▶ 再生` に切り替え
- [ ] `draw()` 内で `image(video, ...)` を呼び、動画フレームを毎フレーム描画

```
確認方法: ボタンを押して動画が再生/停止すること
```

**✅ 完了条件:** ボタンで動画の再生・停止が切り替わる

---

## Step 4: フレーム送り機能（10分）

**ゴール:** 1フレームずつコマ送り・コマ戻しができる

- [ ] `◀ 前` / `次 ▶` ボタンを追加
- [ ] `video.time()` で現在の再生位置を取得
- [ ] ボタン押下でフレーム単位（約 1/30秒）の移動処理を実装
- [ ] 動画が停止中のときだけコマ送りを有効にする

```
確認方法: 停止状態でコマ送りボタンを押し、1フレームずつ動くこと
```

**✅ 完了条件:** フレーム単位での前後移動ができる

---

## Step 5: PoseNetの初期化と接続（10分）

**ゴール:** PoseNetモデルがロードされ、動画を解析対象として設定される

- [ ] `pose.js` を作成
- [ ] `ml5.poseNet(video, modelLoaded)` でモデルを初期化
- [ ] ロード完了時にコンソールに `"PoseNet Ready"` と表示
- [ ] `poseNet.on('pose', callback)` でポーズ検出イベントを登録
- [ ] コールバックで `console.log(results)` して座標データの構造を確認

```
作るファイル: pose.js
確認方法: コンソールに座標データ（JSON）が流れてくること
```

**✅ 完了条件:** コンソールにキーポイント座標が出力される

---

## Step 6: キーポイント（関節点）の描画（10分）

**ゴール:** 動画上に関節点（丸い点）が表示される

- [ ] 検出された `pose.keypoints` を `sketch.js` のグローバル変数に保持
- [ ] `draw()` 内で各キーポイントの `(x, y)` に円を描画
- [ ] 信頼度スコア（`score`）が低い点は描画しない（閾値: 0.2）
- [ ] 点の色・サイズを調整して見やすくする

```
確認方法: 動画再生中に、人体の関節位置に色付きの点が表示される
```

**✅ 完了条件:** 動画上に関節点がリアルタイムで描画される

---

## Step 7: 骨格線の描画（10分）

**ゴール:** 関節点同士を線で繋いで「骨格」として表示される

- [ ] PoseNetのキーポイント接続定義を配列で定義  
  （例: `[["leftShoulder", "leftElbow"], ["leftElbow", "leftWrist"], ...]`）
- [ ] `draw()` 内で各ペアの座標間に `line()` を描画
- [ ] 線の太さ・色を調整（半透明の蛍光色など）
- [ ] 信頼度の低い線は描画しない

```
確認方法: 棒人間のような骨格が動画上に表示される
```

**✅ 完了条件:** 人体の骨格線がリアルタイムで描画される

---

## Step 8: 角度計算ユーティリティの実装（10分）

**ゴール:** 3つの関節点から角度を計算する関数が動作する

- [ ] `pose.js` に `calculateAngle(pointA, pointB, pointC)` を実装
- [ ] ベクトル演算（内積 / 外積）で2辺の間の角度をラジアン→度数に変換
- [ ] テスト用に固定座標値を入れて、期待する角度が返ることをコンソールで確認

```
確認方法: console.log で既知の角度（90°、180°等）が正しく算出される
```

**✅ 完了条件:** `calculateAngle()` が正しい角度を返す

---

## Step 9: 関節角度のリアルタイム表示（10分）

**ゴール:** 画面上に主要な関節角度が数値で表示される

- [ ] 投球に重要な角度ペアを定義:
  - 肩-肘-手首（腕の曲がり）
  - 腰-膝-足首（軸足の曲がり）
  - 肩-腰-膝（体の傾き）
- [ ] 各フレームで `calculateAngle()` を呼び、結果をキャンバス下部に `text()` で表示
- [ ] 角度の値を小数点1位に丸める

```
確認方法: 動画再生中に角度値がリアルタイムで変化すること
```

**✅ 完了条件:** 主要関節の角度がリアルタイムで数値表示される

---

## Step 10: UIの仕上げとスタイリング（10分）

**ゴール:** 見た目が整い、使いやすい状態に仕上がる

- [ ] `style.css` でレイアウトを整理（中央揃え、余白、フォント）
- [ ] タイトルヘッダーの追加
- [ ] ボタン群のスタイリング（サイズ、色、配置）
- [ ] 角度表示エリアのデザイン調整
- [ ] モデルロード中の「読み込み中...」表示を追加

```
確認方法: 全体の見た目がMVP.mdの画面構成イメージに近いこと
```

**✅ 完了条件:** MVP.mdの画面構成に近い見た目になっている
