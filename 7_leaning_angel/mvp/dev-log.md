## Phase 7: チャット音声読み上げ（さくら先輩ボイス）

### Step 7-1: 数式→読み上げテキスト変換（ルビ振り） 🗣️
- [ ] `composables/useVoice.js` を新規作成
- [ ] チャット応答テキストをGemini APIに送り、音声読み上げ用テキストに変換するプロンプトを作成
  - 数式 `$\lambda$` → 「ラムダ」、`$Av = \lambda v$` → 「エーブイ イコール ラムダブイ」
  - 記号 `→` → 「は〜になる」、`∈` → 「に属する」
  - ルビ振り例: 固有値(こゆうち)、行列式(ぎょうれつしき)
- [ ] `prompts/voiceConversionPrompt.js` にルビ振り専用プロンプトを定義
- [ ] Gemini APIで変換し、読み上げ用テキストが正しく生成されることをconsoleで確認

**確認ポイント:** `「Av = λv を満たすベクトル」` → `「エーブイ イコール ラムダブイ を満たすベクトル」` に変換される

---

### Step 7-2: AI音声合成＆再生 🔊
- [ ] Web Speech API (`SpeechSynthesis`) を使って読み上げ用テキストを音声出力
  - MVPではブラウザ内蔵の音声合成を使用（外部API不要で即動作）
  - 声のピッチ・速度を調整して「先輩っぽい声」に近づける
- [ ] 各チャット応答メッセージの横に「🔊」再生ボタンを配置
- [ ] タップ → ルビ振り変換 → 音声再生 の一連のフローを接続
- [ ] 再生中はボタンをアニメーション表示（波形アイコンなど）

**確認ポイント:** チャットの応答メッセージ横の🔊をタップすると、数式が正しい読みで音声再生される

---

### Step 7-3: VOICEVOX連携（オプション強化） 🎤
- [ ] VOICEVOX APIの接続設定（ローカルサーバー or VOICEVOX Web API）
- [ ] Web Speech APIの代わりにVOICEVOXの音声で再生するオプションを実装
- [ ] さくら先輩のキャラクターに合った声質（話者ID）を選定
- [ ] VOICEVOX未起動の場合はWeb Speech APIにフォールバック

> VOICEVOX連携は「声の可愛さ」がコア体験に不可欠と判断した場合のみ実装する。

## Phase 11: 複数記事対応（マルチ単元化）

> MVP では固有値の1記事のみを固定表示しているが、将来的に単元を増やすために
> **ルーティング（URL切替）** と **記事一覧ページ** を導入する。

### Step 11-1: vue-router 導入＆ページ分割 🛤️
- [ ] `npm install vue-router` で Vue Router を追加
- [ ] `src/router/index.js` を作成し、以下のルートを定義
  - `/` → 記事一覧ページ（`ArticleListView.vue`）
  - `/article/:slug` → 個別記事ページ（`ArticleView.vue`）
- [ ] `main.js` にルーターを組み込み、`App.vue` の `<MathArticle />` を `<RouterView />` に置き換え
- [ ] 既存の固有値記事が `/article/eigenvalue` で表示されることを確認

**確認ポイント:** URLを直接入力して `/article/eigenvalue` で記事が表示される

---

### Step 11-2: 記事一覧ページの作成 📋
- [ ] `src/views/ArticleListView.vue` を新規作成
- [ ] `content/` 内の全記事データからタイトル・概要を一覧表示するカード UI を実装
- [ ] カードタップで `/article/:slug` に遷移する
- [ ] 記事データの**メタ情報（slug・タイトル・概要・アイコン）** を管理する `content/index.js` を作成

**確認ポイント:** トップページに記事の一覧がカードで表示され、タップで個別記事に遷移できる

---

### Step 11-3: 記事データの動的読み込み＆2つ目の単元追加 📚
- [ ] `content/` に2つ目の記事データ（例: `determinant.js` — 行列式）を追加
- [ ] `ArticleView.vue` で URL の `:slug` パラメータに応じて記事を動的にインポート
- [ ] 存在しない slug にアクセスした場合の 404 表示を実装
- [ ] 2つの記事が一覧→個別ページで正しく切り替わることを確認

**確認ポイント:** 一覧から「固有値」「行列式」をそれぞれ選べ、正しい記事が表示される
