# MVP提案書：さくら先輩の数学辞書アプリ

MVPの目的は、**「可愛い先輩アバター × 分かりやすい数学解説」** というコア体験が、ターゲットユーザー（数学に挫折しそうな大学生など）に刺さるかどうかを **最短で検証する** ことです。

---

## 🎯 コンセプト＆ターゲット

| 項目                 | 内容                                                                                                                 |
| -------------------- | -------------------------------------------------------------------------------------------------------------------- |
| **ターゲット**       | 大学レベルの数学（線形代数など）を独学しているが、難解な専門書に挫折しそうな学生・社会人                             |
| **コンセプト**       | 「いつでも質問できて、優しく教えてくれる『さくら先輩』とマンツーマンで学ぶ、恋愛シミュレーション風の数学辞書アプリ」 |
| **プラットフォーム** | スマホ最適化されたWebアプリ                                                                                          |

---

## 📦 MVPのスコープ（実装する必須機能）

MVPでは、機能とコンテンツを **極限まで絞り込み** ます。

### 1. コンテンツ（1単元のみ）

- **対象範囲:** 「線形代数：固有値と固有ベクトル」の1ページのみに限定
- **構成:**
  - 記事本文
  - KaTeXによる数式表示
  - JSXGraphを用いた動かせるシミュレーション（ベクトルが伸びる図など）

### 2. UI / デザイン

- **デザインテイスト:** 恋愛シミュレーションゲーム風（ピンクやパステルカラー基調、可愛いフォント、先輩の大きめな立ち絵）
- **記事ヘッダー:** 記事タイトルの右上にのみ「音声解説」ボタンを1つ配置（タップで先輩が記事の概要を読み上げる）
- **数式のコピペ機能:** 各数式の横に「コピー」ボタンを配置し、タップでクリップボードに保存（スマホでの数式入力の手間を省くため）

### 3. AIチャット機能（さくら先輩との対話）

- **フローティングUI:** 画面右下に先輩のアイコンが常駐。タップで画面下部からチャット（ハーフモーダル）が立ち上がる
- **キャラクタープロンプト:** 裏側でLLM（Gemini等）を動かし、「後輩思いで優しく、少し甘えさせてくれるが、数学の解説は正確な大学院生」という人格をシステムプロンプトで徹底する
- **ワンタップ質問ボタン:** チャット入力欄の上に **「アイコンのみ」のボタン** を配置
  - 💡 ＝「意味を教えて」
  - 📝 ＝「具体例を教えて」
  - タップで自動質問

### 4. カメラ質問機能（Vision AI）

- **画像アップロード:** チャット入力欄の左端に「カメラアイコン」を配置
- **手書きノートの読み取り:** ユーザーが撮影した手書きの数式やノートの画像をLLMのVision機能（Gemini 1.5 Pro/Flashなど）に送り、「ここが分からない」と質問できる仕組み

## 🛠️ 技術スタック（MVP特化版）

> 初期開発スピードを最優先した構成です。

| カテゴリ               | 技術                                                                              |
| ---------------------- | --------------------------------------------------------------------------------- |
| **フロントエンド**     | Vue.js+ Tailwind CSS                                                              |
| **数式レンダリング**   | KaTeX                                                                             |
| **シミュレーション**   | JSXGraph または Desmos API                                                        |
| **バックエンド（AI）** | Gemini API（テキスト対話 & 画像認識の両方を高精度にこなせるため最適）             |
| **音声合成**           | VOICEVOX API（さくら先輩の声を生成。MVPでは事前録音した音声ファイルの再生でも可） |
| **ホスティング**       | Vercel または Firebase Hosting（静的ページのホスティングのみ）                    |

## 🚫 MVPで「やらない」こと（V2以降に回す）

| 除外項目                             | 理由・方針                                                                        |
| ------------------------------------ | --------------------------------------------------------------------------------- |
| **ユーザー登録・ログイン機能**       | アプリを開いたらすぐ使えるようにする                                              |
| **学習ログ・データベース保存**       | Firestore等の構築は後回し。ページリロードでチャット履歴は消えてOK                 |
| **Live2Dなどの複雑なアニメーション** | まずは数パターンの静止画（笑顔、困り顔、ウインクなど）をVueで切り替えるだけで実装 |
| **複数分野のコンテンツ作成**         | 「固有値」の1ページがウケてから、他の単元を作り始める                             |
